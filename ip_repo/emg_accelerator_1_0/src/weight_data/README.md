# weight_data

Current hardware assets for `lightcnn1d_int8_s001_internal`.

These files are generated by:

```bash
python3 emg_gesture_vivado/src/gen_int8_assets.py
```

The old Q8.8 exporter was removed. The current flow reads
`lightcnn1d_int8_s001_internal/deployment/encoder/integer_parameters.npz` and
the golden integer traces, then regenerates both `weight_data/` and the
`emg_gesture_vivado/src/**/testdata/` golden files.

## Quantization

- Weights and activations are signed INT8.
- Runtime activation saturation is `[-127, 127]`.
- Bias files are signed INT32 values already in the accumulator domain.
- Requantization uses signed Q15 multipliers plus per-channel right shifts:

```text
acc_bias = sum(int8 * int8) + bias_int32
product  = acc_bias * multiplier_q15
q        = round_half_away_from_zero(product >> right_shift)
out      = saturate_int8(q)
```

ReLU is applied where the network layer requires it: stem, PW1, PW2, and FC0.
DW1, DW2, GAP requant, and FC1 do not apply ReLU.

## Packed Words

Packed ROM files store INT8 lanes:

```text
word[lane*8 +: 8] = lane_value
```

Lane 0 is the least significant byte, which appears as the rightmost two hex
characters in each line.

Current packed widths:

| File | Lanes | Word Width |
| --- | ---: | ---: |
| `stem/weight_packed.mem` | 4 | 32 bit |
| `pw1/weight_packed.mem` | 8 | 64 bit |
| `pw2/weight_packed.mem` | 12 | 96 bit |
| `fc/weight_packed_256.mem` | 16 | 128 bit |

The FC packed filename still contains `256` for compatibility with existing
scripts/IP names, but the current INT8 packed word is 128 bits wide.

## Removed Legacy Files

The previous Q8.8 flow produced `weight_data/raw/` and
`weight_data/export_weights.py`. Those were removed because they described and
regenerated signed int16 Q8.8 assets, not the current INT8/Q15 contract.

Use `manifest.json` for the machine-readable current layout.
