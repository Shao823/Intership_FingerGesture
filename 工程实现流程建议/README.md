# 工程实现流程建议

本目录用于集中保存后续所有工程实现建议、路线图、任务拆解、验证方案和优化策略。

## 归档规则

- 每次形成新的工程建议时，优先整理为 Markdown 文档放入本目录。
- 文件名使用清晰主题，例如 `工程路线图.md`、`FPGA实现计划.md`、`DMA与PingPong设计建议.md`。
- 文档内容默认使用简体中文。
- 代码标识符、模块名、接口名保持英文。
- 每个建议应尽量包含目标、输入、输出、依赖、风险和验证方法。
