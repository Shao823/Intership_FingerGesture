# 端到端手势识别 FPGA 部署工程路线图

## 1. 当前工程背景

当前项目已有可工作的算法链路：

```text
Signal Acquisition
-> Signal Preprocessing
-> 1-D CNN
-> Classification
```

工程目标不是重新设计神经网络算法，而是在 FPGA 上完成低延迟、高吞吐、低功耗、资源可控的部署优化。

目标系统链路：

```text
ADC
-> Preprocess
-> 1-D CNN
-> CNN Accelerator
-> DMA
-> AXI Stream
-> Wireless
-> PC/Game
```

## 4. 分层任务树

### M0-工程基线梳理

任务名称：工程基线梳理  
目标：冻结算法接口、输入输出格式、性能目标。  
输入：现有采集模块、预处理模块、1-D CNN、`PROJECT_STATUS.md`。  
输出：系统规格说明。  
依赖：无。  
难度：中。  
预计工作量：1-2 天。  
交付物：接口表、数据维度、精度目标、延迟目标。  
风险：算法接口和硬件接口未定义清楚。  
验证：PC 端基线在固定测试集上跑通。

### M1-1-D CNN 硬件映射

任务名称：1-D CNN 硬件映射  
目标：把已工作的 1-D CNN 转成 FPGA 友好的计算图。  
输入：PC 端 1-D CNN 模型、权重、激活函数、输入窗口。  
输出：硬件计算图。  
依赖：M0。  
难度：高。  
预计工作量：2-4 天。  
交付物：层级参数表、卷积/池化/全连接映射方案。  
风险：维度、padding、stride 与软件模型不一致。  
验证：C/C++ 定点模型与原始模型逐层比对。

任务名称：定点量化方案  
目标：确定输入、权重、bias、activation、accumulator 位宽。  
输入：训练后权重和样本数据分布。  
输出：定点格式，例如 `ap_fixed<W,I>`。  
依赖：M1。  
难度：高。  
预计工作量：2-3 天。  
交付物：量化报告、准确率损失评估。  
风险：位宽过小导致准确率下降，位宽过大导致资源超标。  
验证：PC 定点仿真准确率接近浮点基线。

任务名称：1-D CNN 卷积核设计  
目标：实现卷积、激活、池化、全连接核心计算。  
输入：定点模型、层参数。  
输出：HLS 或 RTL CNN IP。  
依赖：M1、定点量化方案。  
难度：高。  
预计工作量：4-7 天。  
交付物：`cnn_accel` 源代码、testbench。  
风险：循环展开过度导致 DSP/LUT 超标。  
验证：HLS C simulation、RTL co-simulation、逐层误差检查。

### M2-DMA 与 Ping-Pong Buffer

任务名称：DMA 与 Ping-Pong Buffer  
目标：建立连续流式输入输出，避免 PS 逐点搬运。  
输入：ADC/预处理输出、CNN 输入格式。  
输出：AXI Stream + DMA + 双缓冲数据通路。  
依赖：M1 CNN IP。  
难度：高。  
预计工作量：4-6 天。  
交付物：DMA 配置、Ping-Pong 控制逻辑、buffer 地址表。  
风险：cache 一致性、帧边界错误、DMA 中断丢失。  
验证：环回测试、连续帧压力测试、吞吐计数器。

任务名称：AXI 接口规范  
目标：定义 AXI4-Lite 控制寄存器、AXI-Stream 数据协议。  
输入：CNN IP 接口需求。  
输出：寄存器表和 stream packet 格式。  
依赖：M2。  
难度：中。  
预计工作量：1 天。  
交付物：接口文档。  
风险：软件和硬件字段理解不一致。  
验证：寄存器读写自测、stream `TLAST/TVALID/TREADY` 仿真。

### M3-Vivado 系统集成

任务名称：Vivado 系统集成  
目标：完成 Zynq PS、CNN IP、DMA、BRAM/FIFO、Wireless 接口的 Block Design。  
输入：CNN IP、DMA 方案、板卡约束。  
输出：可综合实现的 Vivado 工程。  
依赖：M1、M2。  
难度：高。  
预计工作量：3-5 天。  
交付物：BD、XDC、bitstream、硬件平台。  
风险：地址映射冲突、时钟复位域错误、时序失败。  
验证：synthesis、implementation、timing summary、route status。

### M4-嵌入式软件与驱动

任务名称：嵌入式软件与驱动  
目标：PS 侧完成配置、DMA 调度、结果读取、无线发送。  
输入：Vivado 导出的硬件平台。  
输出：裸机或 Linux 应用。  
依赖：M3。  
难度：中高。  
预计工作量：3-5 天。  
交付物：SDK/Vitis 工程、驱动封装、测试命令。  
风险：cache 未 flush/invalidate 导致数据错误。  
验证：单帧推理、多帧推理、DMA 中断统计。

### M5-Wireless 与 PC/Game Demo

任务名称：Wireless 与 PC/Game Demo  
目标：将分类结果稳定传到 PC/Game 端。  
输入：分类结果、通信协议。  
输出：无线链路和 Demo 控制。  
依赖：M4。  
难度：中。  
预计工作量：3-5 天。  
交付物：UART/SPI/RF 协议、PC 接收端、Game 控制 Demo。  
风险：无线丢包、延迟抖动、协议无校验。  
验证：误码率、端到端延迟、长时间控制测试。

### M6-性能优化与收敛

任务名称：性能优化与收敛  
目标：达到低延迟、高吞吐、资源可控、时序通过。  
输入：Vivado/HLS 报告、实测延迟。  
输出：优化闭环。  
依赖：M1-M5。  
难度：高。  
预计工作量：5-10 天，持续迭代。  
交付物：资源、时序、功耗、延迟报告。  
风险：优化一个指标恶化另一个指标。  
验证：LUT、DSP、FF、BRAM、Fmax、Latency、Throughput、Power 对比表。

### M7-部署与交付

任务名称：部署与交付  
目标：形成可复现、可演示、可维护版本。  
输入：最终 bitstream、软件、Demo。  
输出：部署包。  
依赖：M1-M6。  
难度：中。  
预计工作量：2-3 天。  
交付物：bit/xsa/elf、运行说明、测试数据、报告。  
风险：环境依赖不清导致无法复现。  
验证：从空工程重建、板级冷启动演示、验收测试清单。

## 5. 并行开发机会

- 算法量化、Vivado BD 框架、SDK 通信框架可以并行。
- CNN IP 设计与 PC 端定点验证可以并行。
- DMA/Ping-Pong 逻辑与 Wireless 协议可以并行，但最终要在嵌入式软件联调阶段合并。
- 文档、测试集、性能脚本应从 M0 开始同步维护。

## 6. 推荐优先级

优先完成 M0、定点量化、AXI 接口规范。先把数据格式、位宽、接口协议固定，再推进 CNN IP 和 DMA/Ping-Pong。Wireless 和 Game Demo 应放在硬件数据通路稳定之后，避免接口变化造成反复返工。
